# AeroWorks Operations Control Center — Implementation Plan

## Repository state
Empty scaffold (README, LICENSE, .gitignore only). No existing app code, so this is a
greenfield Vite + React + TypeScript project using Microsoft's Power Apps Code Apps
conventions (`pac code` generated app structure).

## Implementation Plan — App Shell with Mock Data

### 1. Project scaffolding
- Initialize a Vite React-TS project (`npm create vite@latest . -- --template react-ts`) as
  the base, then layer Power Apps Code App conventions on top.
- Add core dependencies: `react-router-dom`, `@tanstack/react-query`.
- Add dev dependencies: `vitest`, `@testing-library/react`, `@testing-library/jest-dom`,
  `jsdom` for unit tests of business logic.
- Configure strict TypeScript (`tsconfig.json` with `"strict": true`, `noUnusedLocals`,
  `noUnusedParameters`, `noImplicitReturns`, etc.).

### 3. Routing
- `react-router-dom` with routes: `/` (Dashboard), `/assets`, `/inspections`, `/incidents`,
  `/work-orders`, `/ai-triage`, `/admin`.
- `AppLayout` renders a persistent sidebar/nav (`NavSidebar`) + `<Outlet />`.

### 4. Data layer (mock-first)
- Each feature has a typed mock dataset (e.g., `Asset`, `Inspection`, `Incident`,
  `WorkOrder`, `TriageException`) and a small "service" function (e.g.,
  `getAssets(): Promise<Asset[]>`) simulating async fetch, consumed via TanStack Query hooks
  (`useAssets()`, etc.). This mirrors the shape future Dataverse-generated services will
  have, easing later swap-out.
- `QueryProvider` wraps the app with `QueryClientProvider`.

### 5. Business logic + tests
- Extract at least one non-trivial piece of business logic per feature likely to need
  testing (e.g., a `computeAssetHealthStatus`, `isInspectionOverdue`,
  `prioritizeTriageExceptions` helper) into `shared/utils` or feature-local `logic.ts` files.
- Add Vitest unit tests for these helpers (e.g., `inspections/logic.test.ts`).
- Add `npm run test` script.

### 6. ALM/Dataverse preparation (non-functional in this task)
- Add `services/README.md` documenting that Dataverse-generated services will live there via
  Power Platform CLI (`pac code add-data-source`) and must not be manually edited.
- Add `.env.example` placeholder for future Dataverse connection config (no real values).
- Note in top-level README about project structure and future steps.

### 7. Tooling config
- `package.json` scripts: `dev`, `build`, `preview`, `lint`, `test`.
- ESLint config (React + TS recommended) if included by Vite template; keep minimal.
- `tsconfig.json`, `tsconfig.node.json`, `vite.config.ts`.
